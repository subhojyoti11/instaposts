Why LLM
Inference
Is Slow.

Autoregressive
LLMs generate one token at a time. Each token requires a full forward pass through all model layers, reading every weight from VRAM. For a 70B model on an A100, that is 140GB of memory reads per token. At 2TB/s VRAM bandwidth, that is 70ms minimum per token: roughly 14 tokens per second maximum.
Memory-bound
LLM decoding is memory-bandwidth-bound, not compute-bound. The GPU's tensor cores sit mostly idle waiting for weights to load from VRAM. GPU utilization during decoding is typically 20-40% even on the fastest hardware. Compute is not the bottleneck.
The waste
Every token requires the same full model pass regardless of how predictable that token is. If the next token is obviously "the" after "pick up the phone and answer", the model still reads 140GB of weights to confirm it. Simple tokens cost exactly the same as complex reasoning tokens.
The opportunity
If we could predict several tokens ahead cheaply and verify them in one pass of the large model, we could generate multiple tokens per large model pass. This is the core idea behind speculative decoding: use a small model to guess, use the large model to verify in parallel.

How It
Works.

Draft model
A small, fast model generates K candidate tokens autoregressively. For a 70B target model, the draft model might be 7B or even 68M parameters. The draft model runs K forward passes quickly because it is small. Each pass reads much less VRAM.
Verification
The large target model runs a single forward pass over the K candidate tokens in parallel. Unlike generation, verification can process all K tokens simultaneously because the inputs are already known. The target model produces probabilities for all K positions at once.
Acceptance
Each draft token is accepted or rejected based on whether the target model agrees with the draft's choice. If the target model's probability for the draft token exceeds a threshold, the token is accepted. Rejection is sampled stochastically to preserve the exact output distribution of the target model.
Speedup
If the draft model accepts on average K tokens before the first rejection, each large model pass yields K tokens instead of 1. With acceptance rate alpha of 0.7 and K=5 drafts, expected accepted tokens per large model pass is roughly 3.1. Net speedup is 2-3x over standard decoding.

Draft and
Verify Loop.

# Speculative decoding loop (simplified) def speculative_decode(target, draft, prompt, K=5): tokens = prompt while not done(tokens): # Draft: small model generates K candidates candidates = [] for _ in range(K): t = draft.generate(tokens + candidates) candidates.append(t) # Verify: target checks all K in one pass target_probs = target.forward(tokens + candidates) # Accept greedily until first rejection for i, (draft_t, probs) in enumerate( zip(candidates, target_probs)): if accept(draft_t, probs): tokens.append(draft_t) else: tokens.append(resample(probs)) break # next round with new prefix return tokens

05 / 07

Variants
and Tuning.

Draft model choice
The draft model must be from the same model family as the target for high acceptance rates. Llama 3 70B + Llama 3 8B draft achieves 70-80% acceptance. Mismatched families drop acceptance rates below 50%, eliminating the speedup. Self-speculative variants use earlier layers of the same model as the draft.
Medusa
Adds multiple prediction heads to the base LLM instead of using a separate draft model. Each head predicts a different future position simultaneously in a single forward pass of the target model. No separate model to deploy. Medusa-2 achieves 2.3-3.6x speedup on LLaMA and Vicuna models.
EAGLE
Trains a small autoregressive model on the target model's feature space, not token space. The draft model operates on hidden states rather than raw tokens, dramatically increasing acceptance rates. EAGLE-2 achieves 3-5x speedup and is the current state-of-the-art for production speculative decoding.
Lookahead
n-gram lookahead decoding maintains a pool of candidate phrase completions from recent context. Proposes multi-token completions based on patterns seen earlier in the generation. No draft model needed. Cheaper to implement, lower acceptance rates than learned drafts but zero additional VRAM.

06 / 07

In
Production.

Google
Speculative decoding is used in Google's production Gemini inference stack. Google published the original speculative decoding paper in 2023. Internal benchmarks showed 2-3x speedup on Chinchilla 70B for code generation tasks where predictable token sequences are common.
DeepMind
The "Fast Inference from Transformers via Speculative Decoding" paper by Leviathan et al. from Google Brain formalized the approach. Published at ICML 2023. Proved mathematically that speculative decoding produces identical output distributions to standard autoregressive decoding when using the stochastic acceptance criterion.
Hugging Face TGI
TGI ships speculative decoding as a first-class feature via the --speculate flag. Automatically manages the draft model and verification loop. Works with any model pair of the same family. Enabled by default on compatible HuggingFace Inference Endpoints for 2x throughput at no cost increase.
vLLM
vLLM supports draft-model speculative decoding alongside Medusa and EAGLE heads. Combined with PagedAttention for KV cache management, speculative decoding in vLLM reduces time-to-first-token and increases tokens per second simultaneously. The combination is the current state of production LLM serving.

Quick
Reference.

Speculative decoding
Use a small draft model to propose K tokens, then verify all K with one pass of the large target model. Accepted tokens are generated for the cost of the draft. Speedup equals the average number of accepted tokens per target model pass.
Acceptance rate
The fraction of draft tokens accepted by the target model. Ranges from 50% (mismatched families) to 85% (same-family well-tuned drafts). Acceptance rate directly determines speedup: at 80% acceptance with K=5 drafts, roughly 3.3 tokens are accepted per target model forward pass.
Medusa
Multiple prediction heads added to the base LLM for speculative decoding without a separate draft model. Each head predicts a different future position in a single forward pass. 2.3-3.6x speedup. No separate model deployment overhead.
EAGLE
Draft model trained on the target model's hidden states rather than token probabilities. Higher acceptance rates than token-space drafts. EAGLE-2 achieves 3-5x speedup, the current state of the art for speculative decoding quality.
Output equivalence
Mathematically proven to produce identical output distribution to standard autoregressive decoding. The stochastic acceptance criterion ensures rejected tokens are resampled from the corrected distribution. The model output is unchanged, only the speed improves.
