02 / 07

Every Byte
Crosses Four Walls.

Arrive
Packet arrives at NIC, DMA'd into kernel socket buffer. First copy lands in kernel memory, inaccessible to your application.
recv()
Kernel copies from kernel buffer to userspace buffer. Two copies minimum per receive, plus a context switch each way.
CPU cost
Every copy saturates memory bandwidth and burns cache. At 100 Gbps line rate, the copy overhead consumes more CPU than the application logic.
At scale
Syncing 100 GB of gradients across 1,000 GPUs means 100 TB of memory copies per step. The bottleneck is not the network, it is the copy.

03 / 07

sendfile.
mmap. io_uring.

sendfile()
Copies data from page cache directly to a socket, no copy to userspace. nginx uses this. 40x faster than read+write for file serving at high concurrency.
mmap()
Maps kernel memory into userspace address space. Your app reads the same physical pages the kernel wrote. No second copy after the first page fault.
splice()
Moves data between file descriptors using kernel pipe buffers. Zero copy between two FDs without ever passing through userspace.
io_uring
Submits and reaps async I/O through a shared ring buffer with the kernel. No syscall per operation. Reduces context switch overhead by up to 50% on high-IOPS workloads.

# Traditional TCP (4 copies per round trip):
NIC → kernel socket buf # DMA
kernel → userspace buf # recv()
userspace → GPU mem # cudaMemcpy
# Latency: ~100-200 us. CPU in loop.

# RDMA (register once, then zero copies):
ibv_reg_mr(pd, buf, len) # register once
ibv_post_send(qp, &wr) # NIC reads app memory
# remote NIC writes app mem
# Latency: ~1.5 us. CPU not involved.

05 / 07

Three Flavors
of RDMA.

InfiniBand
Purpose-built RDMA fabric. 1-2 μs latency, 400 Gbps per link. Standard interconnect for HPC clusters and AI supercomputers. Used in all major GPU training clusters.
RoCE
RDMA over Converged Ethernet. RDMA semantics on standard Ethernet hardware. Requires lossless fabric (DCB/PFC). Widely deployed in cloud data centers.
iWARP
RDMA over standard TCP/IP. No lossless fabric required. Higher latency than InfiniBand or RoCE. Easiest to deploy in existing networks without fabric changes.
DPDK
Not RDMA, but kernel bypass for Ethernet. Processes packets entirely in userspace, bypassing the kernel network stack. Used in HFT and Cloudflare's DDoS pipeline.

07 / 07

Quick
Reference.

Zero-copy
Avoiding redundant CPU memcpy between kernel and userspace. sendfile(), mmap(), splice(), io_uring, each eliminates one or more memory copies.
RDMA
Remote Direct Memory Access. NIC reads/writes application memory directly. No CPU on either endpoint after registration. ~1-2 μs latency vs ~100 μs for TCP.
InfiniBand
Purpose-built RDMA fabric. Standard interconnect for AI training clusters and HPC supercomputers. 400 Gbps, sub-2 μs round-trip latency.
RoCE
RDMA over standard Ethernet hardware with lossless fabric. Lower cost than InfiniBand. Widely deployed in cloud data centers and private GPU clusters.
NCCL
Uses RDMA for gradient all-reduce in distributed ML training. The reason multi-GPU gradient sync stays fast enough to be worth doing at scale.

Caption
Your GPU cluster spent more time copying data than training the model. Not because the code was slow. Because data had to cross four walls before a single computation could happen. NIC to kernel buffer. Kernel buffer to userspace buffer. Userspace buffer to GPU driver. GPU driver to GPU memory. Four copies. Two context switches. The CPU in the loop at every step. Modern networking hardware moves data at 400 gigabits per second. But the Linux network stack was designed when that was unimaginable. A packet arrives at the NIC, gets DMA'd into a kernel socket buffer, copied to a userspace buffer on recv(), then copied again into GPU memory via cudaMemcpy. At 100 Gbps line rate, the copy overhead consumes more CPU than the application logic itself. In a distributed ML cluster syncing 100 GB of gradients, the bottleneck stops being the network and starts being the copy. Zero-copy is the first half of the fix. sendfile() sends a file directly from the page cache to a socket without ever copying to userspace. nginx uses this and serves files 40x faster than naive read plus write at high concurrency. mmap() maps kernel memory into userspace address space, your app reads the same physical pages the kernel wrote. io_uring submits and reaps async I/O through a shared ring buffer, eliminating the per-operation syscall overhead entirely. RDMA is the second half. Remote Direct Memory Access lets a NIC read from one machine's application memory and write directly to another machine's application memory, with the CPU uninvolved on both ends after initial registration. No kernel network stack. No context switches. No copies. Latency drops from 100 to 200 microseconds for TCP to 1 to 2 microseconds for InfiniBand RDMA. NCCL, NVIDIA's collective communication library, uses RDMA for all-reduce during gradient synchronization. Every large model training run at Google, Meta, and OpenAI depends on this. Snowflake uses RDMA for shuffle operations between compute nodes. High-frequency trading firms use RDMA and kernel bypass to achieve sub-10 microsecond round trips. We didn't make the network faster. We stopped asking the CPU to carry every byte.
