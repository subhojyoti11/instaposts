The Latency
Problem.

Speed of light
Light travels at roughly 200,000 km/s through fiber optic cable after accounting for refraction. A user in Sydney to a server in us-east-1 covers 16,000km. The round-trip is at minimum 160ms, before any queuing or processing.
Real world
Sydney to us-east-1 actual p50 latency is 250-350ms. For real-time voice AI, autonomous vehicle control, and industrial automation, this is unusable. You cannot drive a robot arm with 300ms of round-trip lag.
Bandwidth cost
A single 4K camera generates 25 Mbps of continuous data. A factory with 200 cameras produces 5 Gbps. Sending all of it to the cloud for processing costs thousands of dollars per month in egress fees before any compute.
Compliance
GDPR and data sovereignty laws restrict which countries user data can transit. A German manufacturer cannot legally send production floor video to a US data center without violating EU data residency requirements. Edge processing keeps data local.

03 / 07

What Edge
Computing
Actually Is.

Definition
Processing data at or near its source instead of sending it to a centralized data center. The "edge" is any compute that sits closer to the user or device than a cloud region: CDN PoPs, 5G base stations, factory gateways, retail devices.
Near-edge
CDN edge nodes run request filtering, auth, and personalization without touching the origin. Cloudflare Workers and Lambda@Edge execute code at 300+ global locations. Sub-10ms latency for users regardless of where the origin server lives.
Far-edge
On-premises gateways at factories, retail stores, and hospitals pre-process sensor data locally. AWS Outposts, Azure Stack Edge, and NVIDIA Jetson run inference workloads at the device. Only aggregated results or anomalies are sent to the cloud.
5G MEC
Mobile Edge Computing runs compute directly inside 5G base station infrastructure. AWS Wavelength and Azure Edge Zones embed cloud services in telecom data centers. Applications reach 5G users with single-digit millisecond latency.

The Trade-offs.

Edge wins
Latency, bandwidth cost, offline operation, and data sovereignty. A factory line must continue functioning when the WAN link drops. An autonomous vehicle cannot wait for a round-trip to make a braking decision. Edge makes these possible.
Cloud wins
Scale, complex ML training, global aggregation, and operational simplicity. Training a model on petabytes of sensor data from 10,000 factories requires centralized compute. The cloud is still the right place for heavy analytical workloads.
Complexity cost
Edge computing multiplies your operational surface area. Instead of one cloud region, you manage hundreds or thousands of edge nodes with inconsistent hardware, unreliable network links, and limited remote debugging capability. This cost is real.
The model
Most successful architectures use both. Edge handles real-time inference, anomaly detection, and compliance. Cloud handles model training, historical analytics, and long-term storage. Data flows one-way from edge to cloud for aggregation.

06 / 07

In
Production.

Tesla
Each Tesla vehicle runs inference on 8 custom FSD chips generating 72 TOPS. Vision, path planning, and control decisions are made onboard. Fleet telemetry is sent to the cloud for training, but no vehicle requires cloud connectivity to drive.
Amazon Go
Hundreds of cameras per store feed local GPU servers running computer vision inference. Walk in, grab items, walk out. Item detection and checkout happen at the store edge. Transaction records sync to the cloud after the customer leaves.
AWS Outposts
Physical AWS racks deployed in customer data centers, running the same APIs as us-east-1. Goldman Sachs runs Outposts inside their trading floors for sub-millisecond order routing that cannot tolerate public internet latency.
Siemens
Siemens Industrial Edge runs ML inference on manufacturing lines using NVIDIA Jetson hardware. Defect detection models process camera streams locally. Only defect images and aggregated quality metrics are uploaded to MindSphere, their cloud IoT platform.

07 / 07

Quick
Reference.

Edge computing
Processing data at or near its source rather than in a centralized cloud region. The edge ranges from CDN PoPs and 5G base stations to factory gateways and IoT devices.
MEC
Multi-access Edge Computing: compute embedded in telecom infrastructure. AWS Wavelength and Azure Edge Zones run inside carrier data centers. Applications reach 5G users in single-digit milliseconds from the nearest base station.
CDN edge
Code running at CDN Points of Presence, 10ms from most users worldwide. Cloudflare Workers, Lambda@Edge, and Fastly Compute handle auth, A/B testing, and personalization without reaching the origin server.
Far edge
On-premises hardware at factories, hospitals, and retail stores. AWS Outposts, Azure Stack Edge, and NVIDIA Jetson run inference workloads locally. Only results and anomalies are forwarded to the cloud for aggregation and training.
Latency floor
Light travels 200,000 km/s through fiber. A Sydney to us-east-1 round trip is at minimum 160ms. Processing on an edge server within 50km of the user brings this under 1ms, which is the boundary for real-time control systems.
