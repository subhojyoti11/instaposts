02 / 07

The Anonymization
Fallacy.

The claim
Remove names, emails, and IDs from a dataset and it is private. This is the most common privacy mistake in data engineering, and it has been repeatedly disproven with real datasets.
Re-identification
Latanya Sweeney showed in 2000 that 87% of Americans could be uniquely identified using only ZIP code, birth date, and gender. All three appear in most "anonymized" health records without modification.
Netflix prize
Netflix released a "de-identified" movie rating dataset in 2006 for a machine learning competition. Researchers linked 99% of records to named individuals by correlating with public IMDb ratings using only a handful of movies and approximate dates.
The gap
Anonymization removes direct identifiers but not re-identification risk. Differential privacy addresses the gap by providing a mathematical guarantee that no query result reveals information about any individual in the dataset.

03 / 07

The Formal
Definition.

The guarantee
An algorithm is epsilon-differentially private if its output distribution changes by at most a factor of e^epsilon when any single record is added or removed from the input dataset. The output is statistically indistinguishable whether or not any individual's data is included.
Epsilon
The privacy budget parameter. Epsilon = 0 means perfect privacy (but the output is useless). Epsilon = 1 gives a tight bound. Epsilon = 10 allows more utility but weaker privacy. Lower is stricter.
The key insight
DP does not prevent learning aggregate facts about the population. It prevents learning facts about any individual. The average salary can be released accurately. Whether Alice's salary is above the average cannot be inferred.
Composition
Privacy budgets are consumed with each query. Ten queries each with epsilon = 1 consume a total budget of epsilon = 10. Once the budget is exhausted, no further queries are permitted without risking privacy guarantees.

The Laplace
Mechanism.

# Add calibrated noise to make a count query DP: import numpy as np def dp_count(true_count, epsilon, sensitivity=1): # Laplace noise calibrated to sensitivity / epsilon noise = np.random.laplace( loc=0, scale=sensitivity / epsilon ) return true_count + noise # Example: true count = 1000 users # epsilon = 1.0 → noise scale = 1 result = dp_count(1000, epsilon=1.0) # result ≈ 1000 ± a few, never reveals any individual

05 / 07

DP in
Machine Learning.

DP-SGD
Differentially Private Stochastic Gradient Descent clips per-sample gradients to bound sensitivity, then adds calibrated Gaussian noise before updating model weights. The trained model's weights cannot be used to reconstruct any training example.
Memorization
LLMs without DP training can memorize verbatim training data. Researchers have extracted phone numbers, addresses, and email addresses from GPT-2 using targeted prompts. DP-SGD prevents this class of attack.
The tradeoff
DP-SGD requires more training steps to achieve the same accuracy as non-private training. A model trained with epsilon = 8 typically achieves 95% of the accuracy of a non-private model on the same task, with strong privacy guarantees.
Local DP
In local differential privacy, noise is added on the client device before data is sent. Apple uses this to collect keyboard usage statistics from iPhones. The server never sees an individual's raw data, only pre-noised reports.

06 / 07

In
Production.

Apple
Apple deployed local differential privacy across iOS in 2016 to collect emoji usage, QuickType suggestions, and Safari crash data. The first large-scale production deployment of DP, covering hundreds of millions of devices.
Google RAPPOR
Google uses RAPPOR (Randomized Aggregatable Privacy-Preserving Ordinal Response) in Chrome to collect browser statistics with local DP. Deployed since 2014 across Chrome's user base.
US Census
The US Census Bureau used differential privacy for the 2020 decennial census, adding calibrated noise to published statistics to comply with Title 13 confidentiality requirements. The first national census to use DP at scale.
TensorFlow Privacy
Google's TensorFlow Privacy library implements DP-SGD for Keras models. PyTorch Opacus is the equivalent for PyTorch. Both handle gradient clipping, noise addition, and privacy accounting in a few lines of configuration.

07 / 07

Quick
Reference.

DP definition
An algorithm is epsilon-DP if its output distribution changes by at most e^epsilon when any single record is added or removed. Provides a mathematical guarantee that no individual's presence in the dataset is detectable from the output.
Epsilon
Privacy budget parameter. Lower values give stronger privacy with more noise and less accuracy. Higher values give more accuracy with weaker privacy. Typical production values range from 1 to 10 depending on the sensitivity of the data.
Laplace mechanism
Adds Laplace-distributed noise calibrated to sensitivity / epsilon. Used for numeric queries like counts and sums. The noise scale grows as epsilon shrinks, maintaining the privacy guarantee at the cost of accuracy.
DP-SGD
Clips per-sample gradients then adds Gaussian noise before each weight update during model training. Prevents the trained model from memorizing individual training examples. Implemented in TensorFlow Privacy and PyTorch Opacus.
