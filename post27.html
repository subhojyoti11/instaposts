Output costs
4× more.
Here's why.

GPT-4o per 1M tokens:
INPUT: $2.50 ← processed in parallel
OUTPUT: $10.00 ← generated one by one
──────────────────────────
Each output token must:
load full model from HBM
read entire KV cache
one GPU cycle per token
sequential, can't parallelize
──────────────────────────
"Think step by step" = 4× more output
Great for quality. Expensive to run.

The rule: Every instruction that makes the response longer costs you. "Be thorough," "explain your reasoning," "list all possibilities" all generate more output tokens. Optimize your prompts for concise, targeted responses.

Chat history
compounds
every turn.

10-message conversation:
Msg 1: 500 tok
Msg 3: 500 + all prev = 3,000
Msg 5: 500 + all prev = 7,500
Msg 10: 500 + all prev = 25,000
──────────────────────────
Total: ~75,000 input tokens
Not 5,000. Fifteen times more.
──────────────────────────
+ system prompt (2K) × 10 msgs
= 20,000 extra tokens silently

Prompt caching fixes this. Anthropic: 90% discount on cached tokens. OpenAI: 50%. If your system prompt is identical across requests, enable caching immediately. Most developers don't.

Tokens you
never see
but pay for.

10K+
Reasoning tokens. o3, GPT-5.4 Pro generate "thinking" tokens internally. You never see them. You pay for every one
85-170
Image tokens per tile. Every image input is split into tiles. A 1024×1024 image costs 170 tokens at input rate. 10 images = 1,700 tokens
8,000
Web search tokens. GPT-4o with web search charges a fixed 8K input tokens per search call. Even if you only needed a date
tool
Function definitions. Every tool/function you register eats input tokens. 10 tools × 200 tokens each = 2,000 tokens per request, silently

The lesson: Your actual token consumption is always higher than input + output. Reasoning tokens, image tiles, search content, and tool definitions all add invisible overhead that shows up on the bill.

Languages are
not equal.

English is the most token-efficient language. Everything else costs more for the same content.

"Hello, how are you?" in tokens:
──────────────────────────
English: 5 tokens
Spanish: 7 tokens (1.4×)
Hindi: 12 tokens (2.4×)
Chinese: 8 tokens (1.6×)
Japanese: 10 tokens (2.0×)
Arabic: 15 tokens (3.0×)
──────────────────────────
Same meaning. 3× the cost.

Why: Tokenizers (BPE) are trained primarily on English text. Non-Latin scripts get split into more sub-word pieces. A single Hindi word might become 4-5 tokens while the English equivalent is 1. Newer models are improving this, but the gap persists.

Cut your
token bill
by 70%.

01
Enable prompt caching. 50-90% off repeated system prompts. The single biggest savings available
02
Set max_tokens. Cap responses at 500. Prevents runaway output that no user will read anyway
03
Summarize history. After 5 messages, summarize the conversation instead of sending full transcript
04
Route by complexity. Simple queries to Nano ($0.10/MTok). Complex ones to flagship. 80/20 split saves 70%
05
Batch non-urgent work. OpenAI Batch API = 50% off. Analytics, summaries, classifications
06
Trim tool definitions. Only register tools the current query might need. Don't send all 20 every time

The bottom line: Tokens are the electricity bill of AI. You can't avoid them. But you can engineer your prompts, history management, and model routing to pay for intelligence, not waste.
