03 / 07

Host → Client
→ Server → Tool.

Host (Claude Desktop, Cursor, VS Code)
└─ MCP Client (speaks the protocol)
├─ MCP Server: filesystem
│ └─ read_file, write_file, list_dir
├─ MCP Server: github
│ └─ get_pr, create_issue, push_code
└─ MCP Server: postgres
└─ query, describe_table, explain

The key separation: The Host is the AI app. The Client is the protocol adapter. The Server is the integration. The model never calls your tools directly, it routes everything through MCP.

04 / 07

Three things MCP
servers expose.

Tools
Functions the model can call. Name, description, JSON schema. The model decides when to invoke them. run_query, send_email, search_web
Rsrcs
Data the model can read. Files, DB rows, API payloads, addressed by URI. file://main.py or db://users/42. Fetched on demand, not pushed into context
Prompts
Reusable prompt templates. Pre-built instructions invokable by name. "code-review", "summarize-pr". Consistent every time, baked into the server

Most servers only use Tools. Resources are for browsable data. Prompts are for teams that want standardized AI behaviors. Start with Tools, they cover 90% of use cases.

05 / 07

How messages
actually travel.

Wire format: JSON-RPC 2.0
────────────────────────────
stdio ← local tools (recommended)
Host spawns server as subprocess.
Talks via stdin / stdout.
Zero network. Secure by default.
Used by: filesystem, git, sqlite
────────────────────────────
SSE ← remote tools
Server runs as HTTP service.
Client connects via EventSource.
Multi-client, cross-machine.
Used by: Slack, GitHub API, SaaS

stdio is safer. No tokens over the wire, no open ports. For SSE-based remote tools, treat auth like any API, OAuth 2.0 support is being standardized into the protocol now.

06 / 07

The ecosystem
today.

Files
Filesystem (Anthropic official). Read, write, search files. Most-used server. Ships with Claude Code by default
GitHub
GitHub (official). Read repos, create PRs, comment on issues, push code. Core to every agentic coding workflow
DB
PostgreSQL / SQLite. Query databases, describe schemas, explain query plans. Read-only mode strongly recommended for production
Web
Fetch / Browser. Hit any URL or run JS in a headless browser. Agents can scrape, test UIs, and fill forms
Mem
Memory (Anthropic). Persistent key-value store across sessions. The agent remembers facts about you between conversations

07 / 07

The practical
cheatsheet.

01
MCP ≠ tool calling. Tool calling is model-specific JSON. MCP is a universal protocol, one server works with Claude, GPT-5, Gemini
02
Start with stdio. Local, no network, no auth. Ship in an afternoon. Add SSE when you need remote multi-client access
03
Write good tool descriptions. The model reads the description to decide when to call it. Vague description = wrong calls = broken agent
04
Read-only first. Validate the plan before granting write access. Destructive tools need human-in-the-loop confirmation
05
MCP is not auth. The protocol carries the message. You still own security, what your tool does with the call is your responsibility
