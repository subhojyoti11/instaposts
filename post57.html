02 / 07

The Kernel
Module Trap.

Module
The traditional way to extend Linux. Write C, compile against kernel headers, load with insmod. Live in kernel space with full privileges.
Risk
A bug in a kernel module crashes the whole machine. No sandboxing, no safety net. One bad pointer dereference equals a kernel panic.
Deploy
Modules must be recompiled for each kernel version. Updating often requires a reboot. No live patching at production scale.
Result
Nobody ships custom kernel modules to millions of machines. The blast radius is too large. The deployment pipeline is too fragile.

03 / 07

A Safe VM
Inside the Kernel.

VM
eBPF is a bytecode VM inside the Linux kernel. Programs are compiled to eBPF bytecode and loaded via the bpf() syscall.
Verifier
Before execution, the kernel proves the program is safe: no infinite loops, no out-of-bounds memory access, no possible kernel panic.
JIT
Verified programs are JIT-compiled to native machine code. Runs at near-native speed on x86 and ARM. Safety and performance, both.
Deploy
Bad programs are rejected at load time, the kernel is never at risk. Load, update, and remove eBPF programs without rebooting.

05 / 07

Maps: The
Data Bridge.

Maps
Key-value stores in kernel memory. eBPF programs write data into maps; userspace processes read from them without a syscall per event.
Types
Hash maps, arrays, ring buffers, LRU caches, per-CPU maps, socket maps. Each optimized for a different access pattern and concurrency model.
Ring buf
The preferred way to stream kernel events to userspace. Ordered, low-overhead, no data loss under high event rates. Used for tracing and profiling.
Both ways
Userspace can write to maps too. Push policy or configuration into a running eBPF program at any time, no reload, no reboot.

07 / 07

Quick
Reference.

eBPF
Extended Berkeley Packet Filter. Safe sandboxed VM inside Linux. User-defined programs attached to kernel events. No module, no reboot.
Verifier
Proves safety before execution. No infinite loops, no invalid memory, no kernel crash possible. Bad programs rejected at load time.
JIT
eBPF bytecode compiled to native x86/ARM at load time. Near-native performance with full safety guarantees.
Maps
Shared kernel-userspace key-value store. How eBPF programs export data and receive configuration. No syscall per event.
XDP
eXpress Data Path. eBPF at the NIC driver, earliest packet processing point. Used for DDoS mitigation, L4 load balancing, packet steering.

Caption
Your Kubernetes cluster runs a feature you never deployed. It ships with the Linux kernel. For decades, the only way to add custom behavior to the Linux kernel was to write a kernel module. Kernel modules live in kernel space. They run with full privileges. And when they have a bug, not if, when, they take down the entire machine. No process isolation. No safety net. A bad pointer dereference in a kernel module is a kernel panic. That is why nobody ships custom kernel code to production machines at scale. The risk is too high and the deployment cost is too steep: new kernel version, recompile, reboot. eBPF changes both of those constraints simultaneously. eBPF is a virtual machine built into the Linux kernel. You write a small program, in C, or Rust, or directly in eBPF bytecode, and load it into the kernel via the bpf() system call. Before it ever runs, the kernel's verifier mathematically proves the program is safe: no infinite loops, no out-of-bounds memory access, no way to crash the system. If it passes, it gets JIT-compiled to native machine code and attached to a hook. The kernel is never at risk. The hooks are what make eBPF remarkable. You can attach to any kernel function entry or return. Any userspace function. Any hardware performance counter. But the most powerful hook is XDP, eXpress Data Path, which sits at the NIC driver level, before the kernel even allocates a socket buffer. eBPF programs at XDP inspect, modify, redirect, or drop packets before they touch the network stack, at hardware line rate. Millions of packets per second per CPU core. Data flows out through maps. Maps are key-value stores in kernel memory that eBPF programs and userspace applications share. A ring buffer map streams events to userspace with near-zero overhead. A hash map stores per-connection state that both sides read and write. Cilium replaced iptables and kube-proxy with eBPF. Meta replaced their L4 load balancer with XDP. Cloudflare drops DDoS traffic before it reaches the network stack. Datadog traces distributed systems without a single line of instrumentation code. The kernel became programmable. Nobody had to change the kernel to do it.
