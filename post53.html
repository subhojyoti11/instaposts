Dense Models
Don't Scale.

Dense
Every parameter activates on every token. Llama 3 70B fires all 70 billion weights to predict the next word in "hello."
Cost
Compute scales linearly with parameter count. Doubling model quality requires roughly 10x parameters. That math breaks fast.
175B
GPT-3's dense parameter count. Training cost: ~$5M. Inference cost: ~6x a smaller model. Quality advantage: real but finite.
Ceiling
Hitting GPT-4 quality with a dense model would require parameters in the trillions. No lab could serve that at any reasonable cost.

03 / 07

Route.
Activate. Skip.

# Dense: all weights fire every token
output = FFN(x) # all weights active

# MoE: route to top-K of N experts
scores = Router(x) # N learned logits
top_k = topk(scores, k=2)
output = expert_0(x)*gate_0
       + expert_1(x)*gate_1
# N=8, K=2: 25% of FFN compute/token

Experts are independent FFN layers. The router is a tiny linear projection. Only the top-K experts are computed, the rest are skipped entirely.

04 / 07

The Gating
Function.

Router
A linear layer: hidden_dim to N expert logits. One forward pass produces N routing scores per token, per layer.
Top-K
Usually K=2 from N=8 or N=64 experts. Smaller K means less compute. Larger N means more capacity. Both are tunable.
Gate weight
Softmax of top-K scores becomes the mixing weight. Each selected expert's output is scaled before summation.
Per token
Routing is decided independently for each token. Two tokens in the same sentence can be sent to completely different experts.

05 / 07

The Balancing
Problem.

# Without load balancing:
# router collapses to 1-2 favorite experts
# others get no gradient and die

# Auxiliary balance loss (Switch Transformer):
L_aux = α * sum(f_i * P_i for i in experts)
# f_i = token fraction routed to expert i
# P_i = mean router prob for expert i
# α = small coefficient (~0.01)

Expert collapse is the main MoE training failure. Without the auxiliary loss, the router routes everything to one expert and the others never learn.

Why it wins
MoE decouples capacity from compute. Train on more data with more parameters. Serve at a fraction of the full model cost.

07 / 07

Quick
Reference.

Dense
All parameters active per token. Compute scales linearly with parameter count. No routing overhead.
MoE
N total experts, K active per token. Compute scales with K × FFN_size, not N × FFN_size. Capacity without cost.
Router
Linear projection to N logits. Top-K selection with softmax-weighted output combination. Per token, per layer.
Balance
Auxiliary loss prevents expert collapse. Encourages uniform token distribution so all experts receive gradient and learn.
Drop
Token dropping is a failure mode. When an expert's buffer fills, excess tokens are skipped. Capacity limits must be tuned carefully.
