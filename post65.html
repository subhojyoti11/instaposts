What RPA
Actually Does.

How it works
Robotic Process Automation records and replays UI interactions. A bot clicks buttons, copies values between fields, and fills forms, mimicking a human operating a keyboard and mouse.
The constraint
RPA follows an exact script. If the UI changes by a single pixel, or a field label is renamed, the bot fails. Maintenance cost grows as fast as the automation estate.
No reasoning
RPA has no ability to interpret ambiguous input or decide between options. Every decision branch must be hard-coded in advance. Exceptions route to a human queue.
What it solved
Connecting legacy systems that had no API. RPA automated the swivel-chair work of copying data between enterprise systems by treating the screen as the integration layer.

03 / 07

What AI Agents
Do Differently.

Reasoning
An AI agent reads a goal and determines the steps required to achieve it. It is not replaying a recording. It plans, chooses tools, evaluates output, and adjusts when results are unexpected.
Tool use
Agents call APIs, search the web, run code, query databases, and write files. Each tool returns a result the agent incorporates into its next decision. Tools compose into multi-step workflows.
Memory
Agents maintain context across steps within a session and can store results in external memory. A research agent recalls that a source was already checked and skips it, without being told explicitly.
Delegation
An orchestrator agent can spawn sub-agents for parallel subtasks. One sub-agent summarizes documents while another queries a database. Results are merged before the orchestrator continues.

05 / 07

Where Agents
Win.

Unstructured input
An agent can read a PDF contract, extract terms, cross-reference a database, and flag anomalies. RPA cannot handle variable document layouts or interpret natural language clauses.
Cross-system tasks
An agent orchestrating five tools in sequence replaces a pipeline of RPA bots each handling one system. When one system changes its API, only the tool definition updates, not every step in the script.
Exception handling
When output is unexpected, the agent reasons about it rather than routing to a human queue. A billing agent that sees a duplicate charge investigates and resolves it within the same session.
Scale
A single agent prompt handles thousands of input variants. RPA needs a separate script for each variant. The maintenance burden of an AI automation estate grows much more slowly than an RPA estate.

07 / 07

Quick
Reference.

RPA
Robotic Process Automation: records and replays UI interactions. Brittle to UI changes, requires explicit scripts for every decision branch. Best for stable, structured, repetitive workflows with no variation.
AI agent
An LLM-powered system that plans and executes multi-step tasks using tools. Handles ambiguous input, adapts when output is unexpected, and delegates subtasks to other agents.
ReAct
Reason-Act loop: the agent alternates between thinking about the next step and calling a tool. Tool output is incorporated into the next reasoning step. The loop continues until the goal is achieved.
Orchestrator
An agent that breaks a goal into subtasks and assigns them to sub-agents running in parallel. The orchestrator collects results and synthesizes a final output without handling each subtask itself.
Tool use
Structured API calls the agent makes to interact with external systems. Each tool has a name, description, and typed parameters. The LLM decides when to call a tool based on the task context.
