You Forgot
to Free That.

// C, manual memory management
char *buf = malloc(1024);
process(buf);
// forgot free(buf) → memory leak
free(buf);
use(buf); // use-after-free → crash
free(buf); // double-free → undefined

~70% of CVEs are memory safety bugs. Microsoft, Google, and NSA have all cited this number. GC trades some throughput for eliminating an entire class of vulnerabilities.

03 / 07

Mark
and Sweep.

Phase 1 | MARK
Start at GC roots (stack, globals)
Traverse all reachable objects
Set "live" bit on each one found

Phase 2 | SWEEP
Scan the entire heap linearly
Reclaim anything without "live" bit

Problem: Stop-The-World pause
App threads halt for both phases

Invented by John McCarthy in 1959 for Lisp. Still the conceptual foundation every modern GC builds on.

04 / 07

Most Objects
Die Young.

Eden
New objects born here. ~98% die before the first minor GC. Collected frequently and cheaply.
Surv.
Survived one minor GC. Two spaces like objects bounce between them, aging each round.
Old
Long-lived objects promoted here. Major GC (Full GC) is expensive, but avoidance is the goal.
Meta
Class metadata. Replaced PermGen in Java 8. Sized by the OS, rarely triggers collection.

05 / 07

Garbage
First.

Rgns
Heap split into ~2048 equal chunks. Each region is dynamically assigned as Eden, Survivor, or Old.
Pause
Predictable pause targets (default 200ms). G1 picks the regions with the most garbage first.
Mark
Concurrent marking runs alongside app threads. No full-heap scans, only live objects counted.
Dflt
Java 9+ default GC. Right choice for most server workloads with heaps over 4 GB.

06 / 07

Sub-Millisecond
Pauses.

<1ms
ZGC pause target. Holds even on terabyte heaps. G1 can spike to 200ms at scale.
Ptrs
Colored pointers. 4 metadata bits per pointer encode GC state, no extra object headers needed.
Barrier
Load barriers. On every pointer read, the JIT checks if relocation needed, transparent to app code.
Cost
~15% throughput overhead. Trade throughput for latency. Production-ready in Java 15+.

07 / 07

Quick
Reference.

M+S
Mark-and-Sweep. Stop-the-world. Foundation of all GC. Fragmentation over time.
CMS
Concurrent Mark-Sweep. Deprecated Java 9. Low pause but fragmentation-prone.
G1GC
Region-based. Predictable pauses. Java 9+ default. Best for most workloads.
ZGC
<1ms pauses. Concurrent everything. Java 15+ production. For latency-sensitive services.
Shena.
Shenandoah. Similar to ZGC. Red Hat origin. Concurrent compaction. Available in OpenJDK.
