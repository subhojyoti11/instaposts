02 / 07

The Single-GPU
Ceiling.

O(N)
FlashAttention makes attention O(N) memory. An 80 GB A100 handles ~200K tokens for a large model before hitting its limit.
1M tokens
At 1M tokens, KV alone exceeds 100 GB in float16, more than any single GPU holds. Add weights and activations: impossible.
Data parallel
Standard data parallelism splits the batch across GPUs, not the sequence. More GPUs do not extend the context limit at all.
Need
A new algorithm is required. One that distributes a single long sequence across multiple GPUs, not just the batch.

03 / 07

Sequence Split.
Ring Pass.

Split
Divide the N-token sequence evenly across GPUs. GPU i holds tokens [i*chunk : (i+1)*chunk] of Q, K, and V.
Q local
Each GPU attends only for its local query tokens, but needs K and V from all other GPUs to compute full attention.
Ring
GPUs form a logical ring. K and V blocks rotate around the ring one step at a time, each GPU computes partial attention with the arriving block.
Complete
After N_GPUs ring steps, every local Q has attended to every K/V block in the full sequence. Attention is correct and complete.

The Ring
Algorithm.

# Each GPU: holds Q_i, K_block, V_block
# online softmax state: O, m, l
for step in range(N_GPUs):
  O, m, l = flash_update(Q_i,
                    K_block, V_block,
                    O, m, l)
  K_block, V_block = ring_shift(...)
# O_i = correct full attention for Q_i

ring_shift (communication) overlaps with flash_update (compute). Network latency is hidden inside GPU work, adding more GPUs does not proportionally increase wall-clock time.

05 / 07

Incremental
Softmax.

Problem
Standard softmax needs the full row to compute its denominator. You cannot correctly accumulate partial attention results across ring steps.
Solution
Online softmax tracks running max m and running sum l. Both are updated at each ring step without needing the full row at once.
Rescale
After each new K/V block arrives, the accumulated output O is rescaled using the updated m. The result stays numerically identical to single-GPU attention.
Exact
No approximation. After all ring steps, O holds exactly the correct attention output, as if the full sequence had always been on one GPU.

Quick
Reference.

Seq. par.
Sequence parallelism. Distributing a single long sequence across GPUs. Ring Attention is the leading algorithm for this.
Ring
Logical GPU topology. K and V blocks rotate N_GPUs times. Each step: compute partial attention locally with the current block.
Online
Online softmax (m, l, O) state. Enables exact accumulation of partial attention across ring steps without storing N×N.
Overlap
ring_shift runs concurrently with flash_update. Communication latency is hidden inside local computation, critical for scaling efficiency.
Use when
Sequences too long for a single GPU, even with FlashAttention. Document-level tasks, genomics, long-video, million-token chat.

Caption
The context limit was never about the model. It was about the GPU. FlashAttention made attention memory-efficient on a single GPU. But even with O(N) memory, a single 80 GB A100 runs out somewhere around 200,000 tokens for a large model. At one million tokens, the key and value matrices alone need more than 100 gigabytes. No single GPU holds that. And standard data parallelism, which splits the batch across GPUs, does nothing to help, it never splits the sequence. Ring Attention solves this by distributing the sequence itself. Divide the sequence evenly across a ring of GPUs. Each GPU holds its slice of Q, K, and V. To compute full attention, you need every K and V block, so you pass them around the ring. Each GPU receives a K/V block from its neighbor, computes partial attention between its local queries and that block, then passes the block forward. Repeat for every GPU in the ring. After N_GPUs passes, every query has attended to every key and value in the full sequence. The hard part is correctness. Standard softmax requires the full row of scores to compute the normalizing denominator. You cannot accumulate partial results naively. Ring Attention borrows the online softmax trick from FlashAttention, tracking a running maximum and a running normalizer at each step. The accumulated output is rescaled after every ring pass using the updated running maximum. The final result is numerically identical to computing attention on one GPU with the entire sequence in view. The other half of the design is efficiency. While each GPU computes local attention, the next K/V block is already in transit. Communication and computation overlap, the ring shift is hidden inside the GPU work. Scaling to 64 or 128 GPUs adds more context without proportionally increasing wall-clock time. Eight GPUs extend your context window by 8x. The limit stopped being the GPU. It became the number of GPUs in the ring.
