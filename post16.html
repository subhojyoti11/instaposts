The Hierarchy

Three layers
of memory.

1
SRAM (On-chip)~150 TB/s effective. 128–384 MB. On the die. Near-zero latency. Where KV cache lives in TPU 8i. ~100× pricier than DRAM.
2
HBM3e (Stacked)8 TB/s. 192 GB/chip. DRAM stacked via TSVs on interposer next to GPU die. Short wires = high bandwidth.
3
DRAM (Off-chip, DDR5)~100 GB/s. Up to TBs. Far from processor, narrow bus. The real bottleneck.
SRAM: 150 TB/s ← on the die
HBM3e: 8 TB/s ← stacked next to die
DDR5: 0.1 TB/s ← across the board
─────────────────────────
SRAM is 1,500× faster than DDR5.

The Solution

Stacking memory
vertically.

HBM's breakthrough: stack DRAM directly next to the processor instead of spreading it across a board.

Traditional (DDR5):
[CPU] ─── long traces ─── [DRAM]
narrow bus, high latency
─────────────────────────
HBM Architecture:
┌── DRAM die ──┐
├── DRAM die ──┤
├── DRAM die ──┤ ← TSVs
├── DRAM die ──┤
[GPU]─┴── base die ──┘
interposer · 1024-bit · ~3mm

Why it works → shorter wires = less energy. 1024-bit bus (vs DDR5's 64-bit) = 16× data per cycle. TSVs punch thousands of holes through silicon. Result: 80× more bandwidth.

The Takeaway

The future is
memory,
not compute.

The bottleneck has shifted: We can build 12.6 PetaFLOPS chips. But if we can't feed them data fast enough, those FLOPS sit idle. Memory now defines AI's pace.

Solutions converging:

HBM4: 2× bandwidth, custom base dies
More SRAM: TPU 8i (384 MB), Groq 3 (500 MB)
Processing-in-Memory: compute inside DRAM
Optical interconnects: photons over copper
Chiplets: shorter wires, parallel paths

The next trillion-dollar company won't build the fastest chip. It'll build the fastest pipe between memory and compute.
