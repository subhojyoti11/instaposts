Full Fine-Tuning
Doesn't Fit.

LLaMA 7B
Full fine-tune needs ~56 GB VRAM. Weights (14 GB) + AdamW optimizer states (28 GB) + gradients (14 GB). No consumer GPU holds this.

Optimizer
AdamW stores 2 momentum tensors per parameter. A 7B model = 42 GB just for optimizer state, 3x the size of the model weights alone.
Need
Most of the update is low-rank. Fine-tuning doesn't need to change all W, only a small subspace of each weight matrix actually matters.

03 / 07

Updates Have
Low Rank.

Insight
Weight changes ΔW during fine-tuning have low intrinsic rank. The actual update concentrates in a small subspace of W's full dimensions.
Decompose
Any ΔW can be approximated as A·B, where A is d×r and B is r×k, and r is much smaller than d or k.
Rank r
Typically 4–64. At r=8 with d=k=4096: A·B has 65,536 parameters vs 16.7M for full W. That is a 256x reduction per layer.
Freeze W
Original weights never change. Only A and B are trained. At inference, merge: W' = W + A·B. Zero extra latency.

04 / 07

Two Matrices,
Full Adaptation.

# Standard linear layer:
h = x @ W # W frozen, d×k

# LoRA: low-rank bypass
h = x @ W # frozen base
  + x @ (A @ B) # A: d×r, B: r×k

# r=8, d=k=4096:
# W: 16,777,216 params (frozen)
# A+B: 65,536 params (trained)

A and B initialize so A·B = 0. Training starts identical to the base model, adapters diverge only as fine-tuning progresses.

05 / 07

QLoRA:
4-Bit + Adapters.

NF4
Normal Float 4. Quantize frozen base weights using a 4-bit data type designed for normally-distributed neural network weights. Better quality than int4.
Double quant
Quantize the NF4 quantization constants themselves. Saves ~0.37 bits per parameter on top of NF4, additional memory reduction at near-zero cost.
Paged opt.
AdamW states page to CPU RAM during memory spikes. Prevents out-of-memory crashes during gradient accumulation without reducing batch size.
Result
65B LLaMA on a single 48 GB A100. Near full float16 fine-tuning quality. The 4-bit base model adds negligible regression.

07 / 07

Quick
Reference.

LoRA
Low-Rank Adaptation. Freeze base W. Train A (d×r) and B (r×k). Update ΔW = A·B. Merge at inference for zero extra latency.
Rank r
Controls adapter size and expressivity. Higher r = more trainable params = more capacity. Typical starting point: r=8 or r=16.
Alpha
LoRA scaling: output = W·x + (α/r)·A·B·x. Controls the adapter's contribution magnitude. Usually set to α = 2r.
QLoRA
LoRA on a 4-bit NF4 quantized base. Double quantization + paged optimizer. 8–10x memory reduction vs full float16 fine-tuning.
Targets
Applied to Q, K, V, output projections. Some recipes extend to FFN layers too. More target modules = more capacity, more VRAM.

Caption
You cannot afford to fine-tune a 70 billion parameter model. Not because the technique is hard. Because the hardware doesn't exist. Full fine-tuning requires storing the model weights, the gradients, and two AdamW optimizer momentum tensors for every single parameter. A 7 billion parameter model at float16 needs 14 gigabytes just for the weights. Add optimizer state and you need 56 gigabytes before a single training step runs. For a 65 billion parameter model, that is over 500 gigabytes. No single GPU holds that. Most labs can't either. LoRA changes the question from "how do we store all of this?" to "how much do we actually need to update?" The key observation is that weight changes during fine-tuning have low intrinsic rank. When a model learns to follow instructions or adapt to a domain, the update to each weight matrix does not fill its full dimensional space, it concentrates in a small subspace. LoRA exploits this by decomposing the update ΔW into two smaller matrices: A of shape d×r and B of shape r×k, where r is much smaller than d or k. The original weight matrix is frozen. Only A and B are trained. At rank 8 with a 4096-dimensional projection, that is 65,536 trainable parameters instead of 16.7 million, a 256x reduction per layer. QLoRA pushed this further. Dettmers et al. quantized the frozen base model to 4-bit using NF4, a data type designed for normally-distributed neural network weights. They layered LoRA adapters on top in full float16. They quantized the quantization constants themselves to save additional memory. And they used paged optimizers to prevent out-of-memory crashes during gradient accumulation spikes. The result: fine-tune a 65B LLaMA model on a single 48 GB GPU. The resulting model, Guanaco, matched ChatGPT on MT-Bench benchmarks. It was trained in 24 hours. Every open fine-tune you have used, Alpaca, Vicuna, WizardLM, Mistral Instruct, was trained with LoRA or QLoRA. The technique democratized fine-tuning overnight. We didn't make GPUs larger. We made the update smaller.
