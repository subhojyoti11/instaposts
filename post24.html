What happens when
you hit send.

You press Enter. Here's what fires:
─────────────────────────
1. TOKENIZE (CPU)
"Explain quantum" → [849, 31137]
─────────────────────────
2. PREFILL (GPU, compute-bound)
All input tokens in parallel
Builds internal representations
─────────────────────────
3. DECODE (GPU, memory-bound)
Generate tokens one by one
Each reads ENTIRE KV cache
─────────────────────────
4. DETOKENIZE → stream to you

The bottleneck: Step 3 is where the cost lives. Each output token loads the full model weights + KV cache from HBM. That's why output tokens cost 3-4× more than input across every provider.

04 / 08

Output tokens
cost 4× more.

The most important pricing detail nobody talks about.

GPT-4o pricing per 1M tokens:
─────────────────────────
INPUT: $2.50 ← your prompt
OUTPUT: $10.00 ← the response
─────────────────────────
Why?
Input: processed all at once (parallel)
Output: generated one by one (sequential)
─────────────────────────
Each output token requires:
• Loading full model from HBM
• Reading entire KV cache
• One GPU cycle PER token

Cost hack: "Think step by step" is great for quality. It's also expensive. A 200-token answer becomes 800 tokens. Your cost just 4×'d.

05 / 08

The conversation tax.

Every message includes the ENTIRE chat history. Cost compounds with every turn.

10-message conversation:
─────────────────────────
Msg 1: 500 tokens
Msg 2: 500 + prev = 1,500
Msg 3: 500 + prev = 3,000
Msg 5: 500 + prev = 7,500
Msg 10: 500 + prev = 25,000
─────────────────────────
Total input for 10 messages:
~75,000 tokens (not 5,000)
15× more than you'd expect.

Plus system prompt: Your system prompt (2,000 tokens) is resent with EVERY request. 10 messages = 20,000 extra tokens. Prompt caching (50-90% off) fixes this, but most devs never enable it.

08 / 08

How to cut your
AI bill by 70%.

01
Model routing. Nano/Mini for 80% of queries. Flagship for the hard ones only
02
Prompt caching. Anthropic: 90% off. OpenAI: 50% off. Same system prompt = massive savings
03
Set max_tokens. Chatbot response rarely needs 2K tokens. Cap at 500. Kill runaway output
04
Cache responses. 1,000 users ask the same FAQ? Store the answer. Don't call the API 1,000 times
05
Batch API. Non-realtime work? OpenAI Batch = 50% off all tokens
06
Trim history. Summarize old messages. Don't send the full transcript every turn

The real lesson: AI costs aren't fixed. They're engineering decisions. $5K/month vs $225K/month is architecture, not capability. The model is the same. The bill depends on how you use it.
