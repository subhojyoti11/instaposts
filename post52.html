The HBM
Bottleneck.

HBM
~2 TB/s bandwidth, 40–80 GB capacity. Slow storage where all tensors live. Every matrix read and write goes through here.
SRAM
~20 TB/s bandwidth, 20 MB on A100. On-chip fast memory. Ten times faster than HBM but 2000x smaller.
N²
Standard attention materializes an N×N matrix. At N=8K, that's 128 MB per layer, far larger than all SRAM combined.
Bound
Standard attention is memory-bandwidth-bound. Not compute-bound. The GPU waits on HBM reads, not ALU utilization.

03 / 07

Three Trips to
Slow Memory.

# Standard attention: 3 HBM round trips
S = Q @ K.T # O(N²), write S to HBM
P = softmax(S) # O(N²), read S, write P
O = P @ V # O(N²), read P, write O

# N=8192, float16, 96 layers:
# S per layer = 128 MB
# Total intermediate writes ≈ 12 GB
# All of it avoidable

The compute takes microseconds. The HBM reads take milliseconds. FlashAttention eliminates the reads.

04 / 07

Tile. Stay
in SRAM.

Block Q
Split the Q matrix into tiles small enough for SRAM. Load one Q tile at a time, never the full matrix.
Block K/V
Stream K and V blocks across all Q tiles. For each Q tile, visit every K/V block once and compute partial attention.
Fused
Compute QK^T, softmax, and PV entirely in SRAM for each tile pair. S and P are never written to HBM.
HBM cost
Read Q, K, V once. Write O once. No intermediate materialization. Memory cost is O(N), not O(N²).

05 / 07

Online
Softmax.

# softmax needs full row, can't tile naively
# online softmax: track running max and sum

for each K/V tile j:
  m = max(m_prev, rowmax(S_ij))
  l = e^(m_prev-m)*l_prev
    + rowsum(e^(S_ij-m))
  O = diag(e^(m_prev-m))*O_prev
    + e^(S_ij-m) * V_j

The key insight: rescale the accumulated output at each tile using the updated running max. Numerically stable. No second pass needed.

07 / 07

Quick
Reference.

HBM
High Bandwidth Memory. ~2 TB/s. Default tensor storage. Where standard attention writes all its intermediate matrices.
SRAM
On-chip fast memory. ~20 TB/s. 20 MB on A100. FlashAttention keeps all tile computations here.
Tiling
Split Q, K, V into blocks that fit in SRAM. Fuse S, P, O into one kernel. Never materialize N×N in HBM.
Online
Online softmax enables tiling. Tracks running max and normalizer per tile. Single-pass, numerically stable.
FA-2
FlashAttention-2 (Tri Dao, 2023). ~9x faster than standard PyTorch attention. Used by every major LLM lab.
