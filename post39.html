02 / 06

Seven companies
touch your $5 coffee.

1
Cardholder (You). Taps phone or enters card number. Your device encrypts the card data before it leaves
2
Merchant. The coffee shop. Sends the charge request through their payment integration
3
Payment Gateway (Stripe/Razorpay). Encrypts, validates, routes the transaction. The API layer devs actually touch
4
Payment Processor. Formats the message for the card network. Often bundled into the gateway (Stripe does both)
5
Acquirer (Merchant's Bank). The bank that receives money on behalf of the merchant. Underwrites the risk
6
Card Network (Visa/Mastercard/Rupay). Routes the authorization between acquirer and issuer. Sets interchange rates
7
Issuer (Your Bank). Checks your balance, approves/declines, puts a hold on funds. Sends back YES or NO

What happens
in those 1.8 seconds.

YOU tap "Pay $47.99"
━━━━━━━━━━━━━━━━━━━━━━━━━
1. App encrypts card → sends to Stripe
2. Stripe tokenizes card, checks fraud
(Radar ML model, ~50ms)
3. Stripe sends to Acquirer bank
4. Acquirer forwards to Visa/MC
5. Visa routes to Your bank (Issuer)
6. Issuer checks: balance? fraud? limit?
7. Issuer sends back: APPROVED
━━━━━━━━━━━━━━━━━━━━━━━━━
Same chain, reverse direction.
Auth code reaches the merchant.
You see ✓. Total time: ~1.8 seconds.

Authorization ≠ Settlement. The money doesn't actually move yet. Settlement happens 1-3 business days later in a separate batch process. The merchant gets an IOU, not cash.

04 / 06

Who takes a cut of your $100 purchase.

1.5-3.5%
Interchange fee. Goes to the issuing bank. Varies by card type: debit ~0.5%, credit ~1.8%, rewards cards ~2.5%, international ~3.5%
0.05-0.15%
Network fee. Goes to Visa/Mastercard. They make money on every swipe worldwide. Pure profit at scale
0.4-1.0%
Processor markup. Goes to Stripe/Razorpay. This is their revenue. The only fee you can negotiate

05 / 06

What Stripe actually does for you.

Stripe/Razorpay hide all 7 parties behind one API call. That's their entire value proposition:

// Your code. That's it. 4 lines.
const charge = await stripe.charges.create({
amount: 4799,
currency: 'usd',
source: 'tok_visa',
});

// What Stripe does behind those 4 lines:
// 1. Encrypts + tokenizes card data
// 2. Runs fraud detection (Radar ML)
// 3. Routes to correct acquirer bank
// 4. Formats ISO 8583 message for network
// 5. Handles 3D Secure if required
// 6. Manages retry logic on soft declines
// 7. Returns auth code or decline reason
// 8. Handles settlement batch 24h later
// 9. Manages chargebacks + disputes
// 10. Stays PCI-DSS compliant for you

Without Stripe: You'd need PCI-DSS Level 1 compliance ($50K-500K/yr audit), direct bank integrations, fraud systems, settlement infrastructure, and a legal team. Stripe charges 2.9% to save you all of that.

06 / 06

The payment cheatsheet.

01
Never touch raw card numbers. Use Stripe Elements or Razorpay Checkout. Let them handle PCI compliance. One breach costs more than your entire revenue
02
Authorization ≠ Capture. Auth puts a hold. Capture moves the money. For hotels/rentals, auth first, capture later. Cancel the auth if the customer no-shows
03
Idempotency keys. Network timeouts happen. Without idempotency keys, retries can double-charge customers. Always send one
04
Webhooks over polling. Don't poll for payment status. Use webhooks. Stripe sends events for every state change. Build your system to be event-driven
05
Interchange is non-negotiable. You can negotiate Stripe's markup. You cannot negotiate Visa's interchange. That 1.8% is set by the card network, not your processor

Every tap of "Pay" is a 7-party relay race running in under 2 seconds. You don't need to build any of it. But understanding it makes you a better engineer, a better PM, and a better founder. The payment stack is invisible infrastructure that moves trillions.
