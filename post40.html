02 / 06

Figma.

Design tool that beat Adobe. 4M+ users. Browser-based. Feels like a native app.

Frontend / Rendering
C++ (Canvas Engine) WebAssembly WebGL (GPU Rendering) React + TypeScript (UI)
Backend / Multiplayer
Rust (Multiplayer Server) Node.js (Services) CRDTs (Real-time Sync)
Data / Infra
PostgreSQL Redis AWS Kubernetes

The surprise: Figma's canvas is written in C++, compiled to WebAssembly. It's closer to a game engine than a web app. The UI around the canvas is React. Multiplayer sync was ported from TypeScript to Rust for stability.

03 / 06

Discord.

Real-time messaging for 150M users. Sub-100ms latency. Voice, video, text at scale.

Frontend / Clients
React + TypeScript (Web) Electron (Desktop) Swift (iOS Native) Kotlin (Android Native)
Backend / Core
Python (Services) Rust (Performance Critical) Elixir (Voice/Real-time) WebSocket Gateway
Data / Infra
PostgreSQL (Users) Cassandra (Messages) Redis (Cache/Sessions) GCP + Cloudflare

The surprise: Started in Python, then rewrote hot paths in Rust when they needed microsecond-level performance. Messages are stored in Cassandra (time-series), not Postgres. Elixir handles real-time voice routing for millions of concurrent streams.

04 / 06

Notion.

The everything-workspace. Docs, wikis, databases, projects. Real-time collaboration at massive scale.

Frontend
React + TypeScript Custom Block Editor WebSocket (Live Sync)
Backend
Node.js (API Layer) CRDTs (Collaboration) Custom OT Engine
Data / Infra
PostgreSQL (Core) Redis (Cache) S3 (File Storage) AWS + Kubernetes Cloudflare CDN

The surprise: Everything in Notion is a "block." Each block is a row in Postgres. A single page with 100 elements = 100 database rows. This sounds insane, but it makes the block editor infinitely flexible. Real-time sync uses CRDTs so edits never conflict.

05 / 06

Linear.

Issue tracking that developers actually love. Keyboard-first. Sub-50ms interactions. The anti-Jira.

Frontend
React + TypeScript Sync Engine (Offline-first) IndexedDB (Local Cache)
Backend
Node.js + TypeScript GraphQL API Real-time Sync Protocol
Data / Infra
PostgreSQL Redis GCP Vercel (Marketing)

The surprise: Linear feels instant because it's offline-first. Your entire workspace syncs to IndexedDB in your browser. Every click reads from local storage first, then syncs with the server. That's why it feels faster than Jira, which round-trips every action to the server.

06 / 06

What $1B startups
have in common.

React
4 out of 4 use React + TypeScript. Not Vue, not Svelte, not Angular. React is the default for serious product engineering in 2026
Postgres
4 out of 4 use PostgreSQL. The boring database wins again. Not MongoDB, not DynamoDB. Postgres handles everything from blocks to messages
Rust
Figma + Discord adopted Rust for performance-critical paths. Not the first language, but the one you reach for when Python or Node.js hits a wall
Offline
Linear + Notion sync locally first. Offline-first architecture is why these tools feel instant. The secret is local reads with async server sync
Boring
No exotic stacks. React, Postgres, Redis, Node, AWS/GCP. The magic is in the architecture (CRDTs, sync engines, WASM), not the framework choice

The lesson: $1B startups don't win with exotic tech stacks. They win with boring foundations and exceptional architecture decisions. React + Postgres + Redis is the stack. CRDTs, offline-first sync, and WebAssembly are the differentiators.
