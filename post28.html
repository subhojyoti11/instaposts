02

Temperature controls randomness.

Prompt: "The capital of France is ___"
──────────────────────────
temp = 0: Paris 99.9%
Lyon 0.05%, Marseille 0.03%
→ always picks "Paris"
──────────────────────────
temp = 1: Paris 72%
Lyon 12%, Marseille 8%
→ usually Paris, sometimes not
──────────────────────────
temp = 2: Paris 35%
Lyon 18%, Marseille 15%...
→ anything goes. Chaos mode

Temp 0 = deterministic, factual, predictable. Use for code, math, classification.

Temp 0.7 = balanced. Use for general chat, writing.

Temp 1.5+ = creative, wild, unreliable. Use for brainstorming, poetry.

03

Top-K limits
the candidates.

Before sampling, only keep the K most likely tokens. Discard everything else.

Next token probabilities (sorted):
Paris 40%
Lyon 15%
Nice 10%
Marseille 8% ← cut off at K=3
Bordeaux 5% ← discarded
...200 more ← all discarded
──────────────────────────
K=3: only Paris, Lyon, Nice remain
Then sample from these three
──────────────────────────
K=1: always pick the top token
K=100: barely any filtering

The problem with Top-K: K is fixed regardless of the distribution. Sometimes 3 tokens are clearly right (K=3 works). Sometimes 50 are valid (K=3 cuts too aggressively). Top-P solves this.

04

Top-P is adaptive.

Keep the smallest set of tokens whose cumulative probability exceeds P. The candidate pool resizes dynamically.

Confident prediction (factual):
Paris 92% ← cumulative > 0.9
P=0.9: only 1 token passes
──────────────────────────
Uncertain prediction (creative):
dark 15% + cold 12% + quiet 11%
+ still 10% + deep 9% + vast 8%
cumulative = 65%... keep going...
+ empty 7% + long 6% = 78%...
P=0.9: 12 tokens pass. Wide pool

Top-P adapts to context. When the model is confident, the pool shrinks to 1-2 tokens. When it's uncertain, the pool expands to dozens. This is why Top-P is generally preferred over Top-K in production.

05

What to set in practice.

code
temp=0, top_p=1. Deterministic. Same input = same output. No creativity needed
data
temp=0, top_p=1. Classification, extraction, structured output. Precision over creativity
chat
temp=0.7, top_p=0.9. Natural, varied responses. Human-like. The default sweet spot
write
temp=0.8, top_p=0.95. Blog posts, emails, marketing. Creative but coherent
brain
temp=1.2, top_p=0.95. Brainstorming, poetry, wild ideas. Accept occasional nonsense

OpenAI recommends: Adjust temperature OR top_p, not both. They interact unpredictably when modified together. Pick one knob and leave the other at default.

The real insight: These settings don't make the model "smarter" or "dumber." They change which tokens get selected from the same probability distribution. The knowledge is identical. Only the sampling strategy changes.
